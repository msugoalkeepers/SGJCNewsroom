Setup

How the App Works

Goalkeepers Pressroom is a single HTML file hosted on GitHub Pages. All data — users, stories, assignments, rundowns, calendar events, pitches — is stored in Google Firebase Firestore (a free cloud database). Authentication uses Google Sign-In. There is no server, no monthly cost, and nothing to install.

ComponentWhat it doesCost
GitHub PagesHosts the HTML file at a public URLFree forever
Firebase FirestoreCloud database for all newsroom dataFree (Spark plan)
Firebase AuthGoogle Sign-In for all usersFree
LanguageTool APIPowers the in-app grammar checkerFree (public API)
✅The app runs entirely within free service tiers. No credit card is required for normal operation. The only thing that needs periodic attention is your Firestore rules (see below).

Setup

GitHub — Hosting the App

First-Time Setup
Create Your GitHub Repository
1
Go to github.com and sign in (create a free account if needed).
2
Click "New repository." Give it a name (e.g. SGJCNewsroom). Set it to Public.
3
Upload your index.html file to the repository root.
4
Go to Settings → Pages → set Source to "Deploy from branch" → branch: main, folder: / (root).
5
Click Save. Your URL will be: https://yourusername.github.io/RepositoryName/
6
Share this URL with students — it's the permanent address of your newsroom.
Pushing an Update
Upload a New Version of the App
1
Download the new index.html from Claude.
2
Go to your GitHub repository in your browser. Click index.html in the file list.
3
Click the pencil icon (Edit) → then the upload button to replace the file, or delete and re-upload.
4
Scroll down and click "Commit changes."
5
GitHub Pages updates within 1–3 minutes. Do a hard refresh (Ctrl+Shift+R or Cmd+Shift+R) to see changes.
⚡For faster updates, install GitHub Desktop (free app) which lets you drag-and-drop files without using the browser interface.

Firebase — Database & Auth

First-Time Firebase Setup
1
Go to console.firebase.google.com. Sign in with a Google account.
2
Click "Add project." Give it a name. Disable Google Analytics if prompted — it's not needed.
3
Left sidebar: Firestore Database → Create database → Start in production mode → choose a US region → Enable.
4
Click the Rules tab. Paste the rule below and click Publish.
5
Left sidebar: Authentication → Get started → Enable Google as a sign-in provider.
6
Project Settings (gear icon) → Your apps → click </> (web) → register app → copy the firebaseConfig block.
7
Paste the firebaseConfig into your HTML file (replacing the existing one) and upload to GitHub.
Firestore Rules — Critical

This is the most important Firebase setting. If the rules are wrong or expired, no one can log in. The correct rule to use — paste this exactly into Firebase → Firestore → Rules, then click Publish:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }
⚠This rule never expires. If you used Firebase's "test mode" during setup, the original rule had an expiry date baked in. When that date passed, all logins stopped working. The rule above has no expiry — it simply requires the user to be signed in with Google. Keep a copy of it somewhere safe.

If users suddenly can't log in, the Firestore rules are the first thing to check. Go to console.firebase.google.com → your project → Firestore → Rules and confirm the rule says if request.auth != null.

Inviting Students

Students do not need accounts created for them. They sign in with Google and are placed in a pending queue. You approve them from inside the app. Send them this email:

Subject: Join the Goalkeepers Pressroom Newsroom
Hi [Student Name],

You've been invited to join Goalkeepers Pressroom — the MSU SGJC student newsroom for the Black SoccerLab.

To get started:
1. Go to: https://msugoalkeepers.github.io/SGJCNewsroom/
2. Click "Continue with Google" and sign in with your university Google account.
3. You'll see an "Access Pending" screen — this is normal. I will approve your account shortly.
4. Once approved, you'll receive a notification and can log in to see your assignments.

Your role: [Reporter / Section Editor / Producer]

Questions? Reply to this email. See you in the newsroom!
[Your name]

Approve a Pending Student
1
Log in to the newsroom. Click Team in the left sidebar. A badge shows the number of pending requests.

At the top of the Team page, find the "Pending Approvals" section.

Remove & Reactivate Members

Only the Editor-in-Chief can remove members. Removal is a soft delete — the account is locked but the data and story history are preserved.

Removing a Member
  • On the Team page, each member card has a red "Remove" button (does not appear on your own card)
  • Two confirmation dialogs appear before removal is committed — you cannot do this accidentally
  • The removed user is locked out immediately and receives a notification
  • They see an "Access Removed" screen if they try to open the app
Reactivating by Accident
  • A "Removed Members" section appears at the bottom of the Team page (EIC only)
  • Click ↩ Reactivate to restore them to pending status
  • Then approve them from the Pending Approvals section — assign their role and section again
🛡You can never accidentally remove yourself — the Remove button does not appear on your own team card.

Roles & Permissions

📌The first person to sign in is automatically made Editor-in-Chief. Every subsequent user is pending until approved.
Reporter
Write stories · Submit for review · Submit pitches · Add comments & reactions · See own assignments only
Cannot: see others' assignments (unless co-assigned) · create assignments · approve users
Section Editor
All reporter abilities · Assign stories within their section · Review stories within their section
Cannot: review outside section · publish stories
Producer
Build and manage broadcast rundowns · Assign segments · View all rundowns
Cannot: assign print/digital stories · publish
Editor
Assign all stories · Review all sections · Approve/deny pending users · Change reporter roles
Cannot: publish stories · remove members · change editor roles
Editor-in-Chief
Full access — publish stories · unlock published content · change any role · remove & reactivate members · PDF export
Cannot: nothing — full control

Running the Newsroom

Editor Markup Mode

When a story is In Review, editors and EICs see a ✏ Editor Markup toggle button in the story editor toolbar. Turning it on lets you type changes directly in the story body — everything you type appears in red so the reporter can clearly see what was changed.

  • An orange banner appears at the top of the editor confirming markup mode is active
  • The reporter sees the red text when they open the story
  • Each markup session creates a version in Version History — the reporter can compare using ⇄ Compare
  • Turn markup mode off to return to normal editing
💡Use the Comments panel alongside markup mode — markup shows what changed in the text, comments explain why. Together they give reporters clear, actionable feedback.

Broadcast Rundowns

Go to Rundowns in the sidebar. Producers, Editors, and EICs can create and edit rundowns.

Create a Rundown
1
Click "+ New Rundown." Fill in show name, air date, air time, duration, and producer notes.
2
Click the show pill to open it. Click "+ Add Segment" for each segment.
3
Fill in segment type (VO, SOT, LIVE, READER, etc.), duration, reporter/anchor, intro copy, chyron, outcue.
4
Drag segment rows to reorder. Click any row to view full details in a popup.

Reporters can view rundowns and see their own segments. Only Producers, Editors, and EICs can edit.

Duplicating the App for a New Project

The entire newsroom can be cloned for a new cohort or project in about 15 minutes. Each instance is completely independent with its own database and users.

StepWhat to do
1. New Firebase projectconsole.firebase.google.com → Add project → set up Firestore + Auth (see Section 3)
2. Update firebaseConfigReplace the 6-line config block in the HTML with the new project's credentials
3. Replace app name"Goalkeepers Pressroom" appears in ~8 places — use find-and-replace
4. Replace subtitle"MSU SGJC Student Newsroom for the Black SoccerLab" — update to new tagline
5. Update invite URLThe invite link appears once — update to new GitHub Pages URL
6. Update calendar dataDelete or replace the World Cup 2026 fixtures and base camp arrays in the JS (optional)
7. New GitHub repoUpload the modified HTML as index.html, enable Pages
8. DoneNew instance is live, empty, and ready for a new cohort
💡Ask Claude to make all substitutions in one pass — provide the new name, subtitle, accent color, and GitHub URL and it can produce a ready-to-deploy file in minutes.
